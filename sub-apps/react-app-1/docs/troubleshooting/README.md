## 快速问题定位

### 🆘 紧急问题处理

#### API无法访问 (500错误)
1. 检查后端服务状态: `ps aux | grep node`
2. 查看后端日志: `cat logs/react-app-1-backend.log`
3. 参考: [API 500错误完整诊断](./api-500-error-diagnosis.md)

#### 服务启动失败
1. 检查TypeScript编译: `npx tsc --noEmit`
2. 检查端口占用: `lsof -i :3002`
3. 参考: [后端服务启动问题](./backend-service-issues.md)

#### 端口冲突
1. 查找占用进程: `lsof -i :端口号`
2. 终止进程: `kill -9 <PID>`
3. 参考: [端口冲突解决方案](./port-conflicts.md)

### 🔍 问题分类索引

#### 按错误类型
- **编译错误**: [后端服务启动问题](./backend-service-issues.md) → TypeScript编译错误
- **运行时错误**: [API 500错误诊断](./api-500-error-diagnosis.md) → 服务运行时问题
- **配置错误**: [常见问题汇总](./common-issues.md) → 配置相关问题

#### 按影响范围
- **服务级别**: 后端服务无法启动
- **功能级别**: 特定API接口异常
- **组件级别**: 前端组件渲染错误

#### 按紧急程度
- **🔴 高优先级**: 服务完全无法启动
- **🟡 中优先级**: 部分功能异常
- **🟢 低优先级**: 性能优化问题

## 使用指南

### 1. 问题报告流程
1. **现象描述**: 详细描述错误现象
2. **环境信息**: 记录操作系统、Node.js版本等
3. **重现步骤**: 提供问题重现的具体步骤
4. **错误日志**: 附上完整的错误日志
5. **尝试方案**: 记录已经尝试过的解决方案

### 2. 解决方案查找
1. **快速查找**: 使用文档内搜索功能
2. **分类查找**: 根据问题类型选择对应文档
3. **关键词索引**: 使用错误信息关键词查找

### 3. 文档维护

- **方案更新**: 发现更好的解决方案时更新文档
